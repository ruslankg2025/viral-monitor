# Viral Monitor — План разработки

> Аналог Piratex.ai для локального использования.
> Анализ вирусных видео по URL, транскрипция, разбор структуры, генерация сценариев.

---

## ✅ ЧТО СДЕЛАНО (Фаза 1–5)

### Backend — полная инфраструктура
- [x] **FastAPI** приложение с lifespan, CORS, structured logging (structlog)
- [x] **База данных** — SQLAlchemy 2.0 async + aiosqlite, 5 таблиц:
  - `bloggers` — профили блогеров (platform, username, avg_views, niche)
  - `videos` — видео с X-factor, is_outlier, comment_rate, ai_analysis
  - `scripts` — готовые сценарии
  - `settings` — все настройки в БД (переключаемые через UI)
  - `api_usage_log` — трекинг расходов на каждый AI-вызов
- [x] **X-Factor анализатор** — медиана просмотров, outlier detection, comment rate
- [x] **Парсеры** (заготовки с fallback-цепочками):
  - YouTube — yt-dlp programmatic API, субтитры
  - Instagram — Apify actor → instaloader fallback
  - TikTok — Apify actor → Playwright fallback
  - VK — официальный VK API v5.199
- [x] **AI система** — мульти-провайдерный роутер с fallback:
  - Groq / Llama 3.3 70B — бесплатная категоризация
  - OpenAI GPT-4o-mini / GPT-4o — суммаризация, паттерны
  - Claude Sonnet — анализ видео, генерация сценариев
- [x] **Транскрибер** — YouTube субтитры → AssemblyAI → fallback на описание
- [x] **Планировщик** — APScheduler, инкрементальный парсинг каждые N часов
- [x] **REST API** — 20+ endpoints (bloggers, videos, analysis, scripts, settings, costs)
- [x] **Cost tracking** — каждый AI-вызов логируется с токенами и стоимостью

### Frontend — базовый UI
- [x] **React 18 + Vite + TailwindCSS** — тёмная тема (#0a0a0a / акцент #e5e000)
- [x] **Layout** — sidebar 240px, навигация, кнопка "Обновить всё"
- [x] **Dashboard** — лента видео с фильтрами (платформа, период, X-factor, CR)
- [x] **VideoCard** — thumbnail, X-Factor badge (серый/оранжевый/красный + pulse), метрики
- [x] **Bloggers** — CRUD блогеров, импорт CSV, карточки с аватарами
- [x] **VideoDetail** — страница видео с метриками, кнопка анализа, Timeline структуры
- [x] **Scripts** — список сценариев, ScriptEditor с редактированием и экспортом TXT
- [x] **Settings** — все API ключи, провайдеры, параметры, CostDashboard с таблицей
- [x] **Toast, ConfirmDialog, EmptyState, Skeleton** — базовые UI компоненты

## ❌ ЧТО НЕ СДЕЛАНО — КРИТИЧЕСКИЕ ПРОБЕЛЫ

### 🔴 ПРИОРИТЕТ 1 — Главная фича (аналог Piratex)

**Страница "Разбор" — ввод URL → полный pipeline → результат**

По скринам Piratex это ЯДРО продукта:
1. Поле ввода URL любого видео (Instagram Reels, YouTube Shorts, TikTok)
2. Кнопка "Разобрать" → запускает полный pipeline:
   - Парсинг метаданных видео по URL
   - Транскрипция (AssemblyAI / YouTube субтитры)
   - AI-анализ структуры (Claude)
   - Генерация вариантов хуков (5 штук с психологическим разбором)
   - Генерация описания к рилсу
3. Результат — готовый сценарий для телесуфлёра
4. 5 вариантов хуков с:
   - Текстом хука
   - Техникой (контраст, угроза времени, факт+дата...)
   - Психологическим механизмом
   - Оценкой: Сильный / Лучший выбор / Альтернатива
   - Примерным хронометражем
5. Описание к рилсу (готовое к публикации)
6. Кнопки улучшения: `+ Улучшить` `Усилить зацепку` `Сократить`
   `Добавить конкретики` `Переписать начало` `Упростить`
7. Поле чата: "Напишите, что изменить..."
8. Кнопки: `Новый разбор` `Скопировать ссылку`
9. История разборов — "Мои видео" (карточки с превью + stats + CR)

**Это отсутствует полностью. VideoDetail — не то же самое.**

## 📋 ПЛАН ПО ФАЗАМ

### Фаза A — API ключи и онбординг (1–2 часа)
```
1. Онбординг-экран при первом запуске (нет ни одного ключа)
2. Валидация ключей при сохранении (реальный test-запрос к API)
3. Индикаторы статуса провайдеров в Settings
4. .env → Settings синхронизация
```

### Фаза B — Backend pipeline для разбора по URL (2–3 часа)
```
1. POST /api/analyze-url
   - Определение платформы по URL
   - Парсинг через соответствующий парсер
   - Сохранение видео в БД
   - Возврат video_id для polling

2. POST /api/videos/{id}/generate-hooks
   - Генерация 5 вариантов хуков
   - Каждый: текст + техника + психологический механизм + оценка + хронометраж

3. POST /api/videos/{id}/improve
   - Принимает action: усилить_зацепку | сократить | конкретики | переписать_начало | упростить
   - Принимает custom_prompt для свободного редактирования
   - Возвращает обновлённый сценарий

4. Обновить промпты в ai/prompts.py:
   - generate_hooks: 5 хуков с психоанализом
   - improve_script: улучшение по команде
```

### Фаза C — Frontend страница "Разбор" (3–4 часа)
```
1. AnalyzePage (/analyze) — URL-инпут как на Piratex
   - Большое поле ввода по центру
   - Счётчик использования "Использовано X из Y"
   - Иконки: Вставьте ссылку → ИИ анализирует → Готовый сценарий

2. AnalyzeResult (/analyze/:id) — страница результата
   - Шапка: платформа + автор + длительность + просмотры + лайки
   - Кнопки: "Новый разбор" | "Скопировать ссылку"
   - Блок "Сценарий для телесуфлёра" с кнопкой Копировать

3. HookVariants — 5 вариантов хуков
   - Каждый: номер + текст + техника + психологический механизм
   - Бейджи: Сильный / Лучший выбор / Альтернатива
   - Хронометраж
   - Кнопка "Использовать" / "Используется"

4. ImprovementBar — панель улучшений
   - Кнопки: + Улучшить | Усилить зацепку | Сократить |
             Добавить конкретики | Переписать начало | Упростить
   - Поле чата под кнопками

5. ReelDescription — описание к рилсу
   - Готовый текст + кнопка Копировать

6. MyVideos (/my-videos) — история разборов
   - Фильтры: Все / В обработке / Готовые / С ошибкой / Лайкнутые
   - Карточки с превью, статусом, CR, кнопками действий
```

### Фаза D — Интеграция и тест (1–2 часа)
```
1. E2E тест полного pipeline с реальным YouTube URL
2. E2E тест с Instagram (Apify или instaloader)
3. Проверка cost tracking
4. Проверка fallback (нет Groq → GPT-4o-mini → Claude)
```
